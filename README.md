# IoT Backend API

Backend API for a smart-device management dashboard. Built with **Go + Echo + PostgreSQL (raw SQL, no ORM)** using clean architecture.

## Features

- JWT authentication with bcrypt-hashed passwords
- Role-based access (`admin`, `user`)
- Location-scoped device isolation — users only see devices in locations they can access; admins see all
- CRUD for devices (`/devices`)
- `GET /me` and `GET /user` return current profile + accessible locations
- Request validation with `go-playground/validator/v10`
- API logging (structured JSON access log via Echo middleware)
- Prometheus metrics at `/metrics`
- Swagger UI at `/swagger/index.html`
- Docker setup for API + Postgres

## Folder structure

```
.
├── app/         # HTTP layer: router, handlers, middleware, validator, error mapping
├── service/     # Business logic: auth, device, JWT
├── repo/        # Data access (raw SQL against Postgres)
├── models/      # DTOs and domain types
├── database/    # Connection pool, schema.sql, migration & seed
├── docs/        # Generated by `swag init`
└── main.go
```

## Requirements

- Go 1.23+
- PostgreSQL 13+
- (Optional) Docker 24+

## Running with Docker (recommended)

```bash
cp .env.example .env
docker compose up --build
```

The API will be available at `http://localhost:8080`.

## Running locally

```bash
cp .env.example .env                       # edit DB credentials if needed
# start Postgres somewhere the .env points to
go mod tidy
go run .
```

The application will:
1. connect to Postgres,
2. execute `database/schema.sql` (idempotent — safe to re-run),
3. seed two default users the first time it runs.

## Default credentials (seeded automatically)

| Role  | Email                 | Password  | Locations |
|-------|-----------------------|-----------|-----------|
| admin | admin@example.com     | admin123  | all       |
| user  | jakarta@example.com   | user123   | Jakarta   |

## Generate Swagger docs

```bash
go install github.com/swaggo/swag/cmd/swag@latest
swag init -g main.go -o docs
```

Then open `http://localhost:8080/swagger/index.html`.

## Endpoints (summary)

| Method | Path                    | Auth | Description                     |
|--------|-------------------------|------|---------------------------------|
| POST   | /api/v1/auth/login      | -    | Login, returns JWT              |
| POST   | /api/v1/auth/register   | admin| Create a user (admin only)      |
| GET    | /api/v1/me              | yes  | Current user profile            |
| GET    | /api/v1/user            | yes  | Alias for `/me`                 |
| GET    | /api/v1/devices         | yes  | List devices (location-scoped)  |
| GET    | /api/v1/devices/:id     | yes  | Device details                  |
| POST   | /api/v1/devices         | yes  | Create device                   |
| PUT    | /api/v1/devices/:id     | yes  | Update device                   |
| DELETE | /api/v1/devices/:id     | yes  | Delete device                   |
| GET    | /health                 | -    | Liveness                        |
| GET    | /metrics                | -    | Prometheus metrics              |

## Example curl

```bash
# Login
TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"admin@example.com","password":"admin123"}' \
  | jq -r .data.access_token)

# Profile
curl -s http://localhost:8080/api/v1/me -H "Authorization: Bearer $TOKEN"

# List devices
curl -s http://localhost:8080/api/v1/devices -H "Authorization: Bearer $TOKEN"

# Create device (Jakarta location)
curl -s -X POST http://localhost:8080/api/v1/devices \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Sensor X","location_id":"11111111-1111-1111-1111-111111111111","status":"online"}'
```

## Location isolation

- Admins see / modify devices in any location.
- Regular users can only see / modify devices in locations present in the `user_locations` table for their account.
- Attempts to read or mutate a device in a non-accessible location return `403 Forbidden`.
