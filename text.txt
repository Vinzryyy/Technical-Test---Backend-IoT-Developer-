Technical Test - Backend IoT Developer 
Design a CRUD for User & Device Access Control API. 
You’re building the backend for a dashboard used by internal staff to manage smart devices.  
You should build your API using Golang, Fiber, and PostgreSQL (no ORM preferable) with clean 
architectured code 
Task 
Your task is to build a simple backend API that handles: 
● Secure login 
● Role-based access to protected resources 
● CRUD for managing "devices" 
Your API also should handles 
1. Authentication 
○ Login endpoint using email + password 
○ Use JWT for authentication 
○ Store users in a DB (PostgreSQL) 
○ Passwords should be hashed (e.g., bcrypt) 
○ GET /me or GET /user - get personal information (role, etc.) 
○ Each user has their own location access 
2. Device Management 
Create RESTful endpoints for: 
○ GET /devices – list all devices 
○ GET /devices/:id – view device details 
○ POST /devices – create a device 
○ PUT /devices/:id – update device 
○ DELETE /devices/:id – delete a device 
○ Each device has only 1 location 
○ Isolate devices data with the user’s location access 
Device fields: 
id, name, location, status (online/offline), updated_at 
And other table on database 
3. Should have validation and error handling 
4. Bonus points 
○ API Logging 
○ API Monitoring 
○ Provide Docker setup to handle API and Database 
Delivery 
You need to deliver: 
● Source code link (in github with public access) 
● Postman collection or Swagger (preferable) docs 
● SQL schema for PostgreSQL (you can put it inside the github) 
● Short README.md with instructions to run 
● A brief presentation to present on user interview (just show the code and doc is enough) 
P.S. No pressure to finish everything — we prioritize clear, structured, working code over full 
features. Good Luck!
